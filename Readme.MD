# TaskFlow — Канбан-доска

Полноценное full-stack приложение для управления задачами.
## Online версия - <https://task-flow-silk-delta.vercel.app/>

---

## Запуск проекта

### Требования

- Node.js 20+
- PostgreSQL 15+ (или Docker)

### Локальный запуск

#### Бэкенд

```bash
cd backend
cp .env.example .env
npm install
npm run start:dev
# API: http://localhost:3001
# Swagger: http://localhost:3001/api/docs
# Open API: http://localhost:3001/api/docs.json
```

#### Фронтенд

```bash
cd frontend
cp .env.local.example .env.local
npm install
npm run dev
# Приложение: http://localhost:3000
```

### Docker запуск

```bash
# Из корня проекта
cp backend/.env.example backend/.env
cp frontend/.env.example frontend/.env
docker compose up --build
```

Приложение на http://localhost:3000
Swagger на http://localhost:3001/api/docs

## Возможности

- **Kanban-доски с drag & drop** — перемещение задач между колонками, сортировка задач и переупорядочивание колонок.
- **Статусы выполнения задач** — отметка задач выполненными из карточки или модального окна с сохранением даты выполнения.
- **Аналитика выполненных задач** — графики по дням, неделям и месяцам, счетчик за сегодня и сводка выполнения в срок/после срока.
- **Очередь несохраненных изменений** — офлайн-изменения задач и перемещения попадают в локальную очередь; после восстановления связи пользователь выбирает, применить их или отменить.
- **Шаблоны досок** — при создании можно выбрать пустую доску или готовый Scrum-шаблон с колонками `Backlog`, `Sprint`, `In Progress`, `Review`, `Testing`, `Done`.
- **Приветственная доска** — при регистрации создается локализованная демо-доска с колонками, открытыми задачами и выполненными задачами с датами относительно времени регистрации.
- **Совместная работа** — доской можно поделиться с другим пользователем, а владелец может управлять участниками.
- **Real-time синхронизация** — WebSocket-события обновляют доски и задачи без ручного refresh.
- **Карточки задач с деталями** — приоритеты, дедлайны, метки, ответственный, описание и визуальное состояние выполнения.
- **Локализация** — интерфейс, шаблоны и демо-доска поддерживают русский и английский языки.
- **Профиль и активные сессии** — пользователь видит свои сессии и может завершить неиспользуемые.
- **Темная и светлая темы** — переключение темы с сохранением выбора пользователя.

### Стратегии рендеринга

| Страница             | Стратегия                  | Причина                                              |
| -------------------- | -------------------------- | ---------------------------------------------------- |
| `/`                  | **Dynamic shell / static-ready content** | Локаль определяется из cookies/headers; приватных данных нет |
| `/auth/login`, `/auth/register` | **Dynamic shell / client-first form** | Метаданные серверные, auth flow клиентский |
| `/boards`            | **Dynamic SSR + TanStack hydration** (`force-dynamic`, `revalidate = 0`, `force-no-store`) | Персональный список грузится на сервере без cache и гидрируется в клиент |
| `/boards/[id]`       | **Dynamic SSR + tagged board cache** (`force-dynamic`, `revalidate = 0`, `force-no-store`) | Access check no-store; общий контент доски кешируется по `board-content:{id}` и инвалидируется backend webhook |
| `/profile`           | **Dynamic SSR shell** (`force-dynamic`, `revalidate = 0`, `force-no-store`) | Защищенные session data |

### Управление состоянием

| Стейт                            | Инструмент            | Причина                                        |
| -------------------------------- | --------------------- | ---------------------------------------------- |
| Серверные данные (доски, задачи) | **TanStack Query**    | Кэширование, фоновый refetch, мутации          |
| Текущий пользователь             | **Zustand AuthStore**    | Нужен везде глобально, реагирует на вход/выход |

## WebSocket события

Все события требуют JWT аутентификацию через httpOnly cookies.

### `board:join` — Подписаться на доску

```javascript
socket.emit('board:join', { boardId: 'xxx' });
```

Подписывает клиента на обновления доски, отправляет текущее состояние.

### `task:update` — Обновить задачу (real-time)

```javascript
socket.emit('task:update', {
  boardId: 'xxx',
  taskId: 'yyy',
  changes: { title: 'New title', priority: 'high' }
});
```

Обновляет данные задачи. Трансляция всем клиентам доски.

### `task:move` — Переместить задачу

```javascript
socket.emit('task:move', {
  boardId: 'xxx',
  taskId: 'yyy',
  columnId: 'zzz',
  order: 2
});
```

Перемещает задачу в другую колонку с автоматическим переупорядочиванием.

## Очередь несохраненных изменений

Для task-мутаций через WebSocket (`task:update`, `task:move`, `task:reorder`) на фронтенде есть in-memory очередь несохраненных изменений.

Если пользователь пытается изменить задачу, отметить ее выполненной или переместить ее во время потери соединения, приложение:

1. делает optimistic update;
2. откатывает UI, если нет надежного socket-соединения или не пришел ack от сервера;
3. не оставляет событие в скрытом socket.io buffer;
4. сохраняет payload изменения в локальную очередь;
5. после восстановления соединения показывает popup с выбором: применить несохраненные изменения или отказаться.

Очередь не переживает перезагрузку страницы. Это сделано намеренно: старые действия не должны применяться без явного согласия пользователя после refresh или новой сессии.

## Аутентификация

### JWT токены

- **Access token** — 15 минут
- **Refresh token** — 7 дней
- Оба в **httpOnly cookies** (защита от XSS)

### Ротация

При истечении Access токена:

1. Клиент отправляет Refresh токен
2. Сервер выдаёт новую пару
3. Старый Refresh токен отзывается

### Детекция повторного использования

Если отозванный Refresh токен используется снова — все сессии пользователя отзываются.

## Особенности проекта

1. **httpOnly cookies** — токены недоступны через `document.cookie`, защита от XSS.
2. **Ротация refresh-токенов** — каждый рефреш выдаёт новую пару и отзывает старый токен.
3. **Детекция повторного использования** — повторное использование отозванного токена отзывает все сессии.
4. **WebSocket ack + rollback** — task-мутации ждут подтверждения сервера и откатывают optimistic update при ошибке.
5. **Очередь несохраненных изменений** — task-мутации, которые не получили успешный WebSocket ack из-за потери соединения, попадают в локальную очередь; после восстановления связи пользователь выбирает, применить их или отказаться.
6. **Аналитика выполнения** — backend отдает daily/weekly/monthly/summary статистику с проверкой доступа и фильтрами по доске/датам.
7. **Шаблоны досок** — backend создает пустую доску или Scrum-доску с преднастроенными колонками.
8. **Приветственная демо-доска** — регистрация пользователя автоматически создает локализованную доску с тестовыми задачами и completed-историей для графиков.
9. **Шаринг досок** — совместный доступ к доске по email или ID пользователя.
10. **Real-time синхронизация** — WebSocket события для изменения позиции и перемещения задач между колонками.
11. **Мультиязычность** — поддержка English и Russian с автоматической детекцией локали через cookies и Accept-Language.
12. **Zustand + TanStack Query** — чёткое разделение серверного стейта и UI-стейта.
13. **Protected route freshness** — `/boards` рендерится динамически и грузит персональный список без cache; `/boards/[id]` делает no-store access check и кеширует общий контент доски по tag, который инвалидирует backend.
14. **FSD-архитектура** — слои `app / widgets / features / shared` с однонаправленными зависимостями.
15. **NestJS guards + decorators** — `@UseGuards(JwtAuthGuard)` + `@CurrentUser()`.
16. **Swagger/OpenAPI** — интерактивная документация API на `/api/docs` с возможностью тестирования.
17. **Генерация типов из API** — TypeScript типы фронтенда синхронизированы с OpenAPI Swagger.
18. **Docker Compose** — полная развёртка всех компонентов (PostgreSQL, Backend, Frontend) одной командой.
